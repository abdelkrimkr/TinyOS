## 2024-05-22 - Improving Developer UX in Headless Environments
**Learning:** In a headless/serial-only environment, standard "GUI" feedback is impossible. UX must focus on text clarity, color coding (ANSI), and explicit instructions for environment-specific interactions (like QEMU hotkeys).
**Action:** Always include "how to exit" instructions in the halt loop for headless kernels to prevent user frustration.
